# Git Commit Message Generator

現在の変更内容に基づいて、適切なコミットメッセージを生成してください。

## 実行手順

1. **Issue番号の取得**（必須）
   - 現在のブランチ名から #XXX パターンを抽出
   - 見つからない場合は「Issue番号を入力してください（番号のみ）:」と確認
   - Issue番号なしでは絶対に進まない

2. **変更内容の分析**
   ```bash
   git diff --cached --stat || git diff --stat
   git diff --cached || git diff
   ```

3. **メッセージ生成**
   以下の3つの形式でメッセージ候補を日本語で提示:
   
   **Option 1: 簡潔版**
   ```
   [動詞]: [簡潔な説明] #[Issue番号]
   ```
   
   **Option 2: 詳細版**
   ```
   [動詞]: [概要説明] #[Issue番号]
   
   - [具体的な変更点1]
   - [具体的な変更点2]
   - [影響や理由]
   ```
   
   **Option 3: Conventional Commits形式**
   ```
   [type]([scope]): [description] #[Issue番号]
   
   [詳細説明]
   ```

4. **使用方法の提示**
   コピー&ペースト可能なコマンドを提供:
   ```bash
   git add . && git commit -m "[メッセージ] #[Issue番号]"
   ```

## メッセージ作成のガイドライン

- 命令形（動詞で開始）を使用
- 1行目は50文字以内を推奨
- 何を変更したかを具体的に記述
- 必要に応じて理由も含める
- **必ず末尾に #Issue番号 を付与**

## 重要事項

- このコマンドはメッセージ生成のみ（コミット実行はしない）
- Issue番号は絶対に省略しない
- 複数のメッセージ形式を提供し、ユーザーが選択
- 実行はユーザーの責任で行う

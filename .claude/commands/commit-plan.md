# Git Commit Plan Generator

現在のGitリポジトリの変更を分析し、論理的なコミット計画を作成してください。

## 実行手順

1. **現在の状態確認**
   - `git branch --show-current` で現在のブランチを確認
   - `git status --short` でファイル状態を確認
   - `git diff --stat` で変更の統計を取得
   - `git diff` で詳細な変更内容を分析

2. **Issue番号の抽出**
   - ブランチ名から #XXX パターンを検出
   - 見つからない場合は「作業中のIssue番号を入力してください（番号のみ）:」と確認

3. **変更の分析とグループ化**
   以下のカテゴリで分類:
   - feature: 新機能追加
   - fix: バグ修正
   - refactor: コード改善
   - docs: ドキュメント変更
   - test: テスト追加・修正
   - style: フォーマット修正
   - chore: 依存関係・設定変更

4. **コミット計画の生成**
   各コミットグループに対して以下を含む計画を日本語で作成:
   - コミットの目的と概要
   - 含まれるファイル一覧（変更行数付き）
   - 推奨コミットメッセージ（必ず #Issue番号 を末尾に含める）
   - 実行用のgitコマンド

5. **依存関係と注意事項**
   - コミットの推奨順序を明示
   - ビルドの安定性を考慮した順序付け

## 重要なルール

- これは読み取り専用の分析です（git add/commit は実行しない）
- すべてのコミットメッセージに必ず #Issue番号 を含める
- 論理的な単位でコミットを分割する
- 各コミットが単独でビルド可能であることを確認
- ユーザー向けの出力は日本語で、コード・コマンドは英語

## エラー処理

- 変更がない場合: 「変更が検出されませんでした」と報告
- main/developブランチの場合: 作業ブランチ作成を推奨
- 変更が多すぎる場合（20ファイル以上）: 段階的なコミットを推奨

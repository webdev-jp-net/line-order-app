# UI/UXデザイン表記規約

## 概要
本ドキュメントは、UIデザインにおける表記の統一性を保つための規約を定めたものです。
ユーザー体験の一貫性を確保し、混乱を防ぐことを目的としています。

## 基本原則

### 文体の統一
- **敬語レベル**: 口語（です・ます調）を基本とする
- **表記**: ひらがな・カタカナ・漢字のバランスを考慮
- **トーン**: 親しみやすく、専門用語は最小限に

### 表記揺れの防止
一貫性のある表記を使用し、同一画面・同一機能では必ず同じ表記を使用する。

## 主要な表記規約
### アクション関連

| 推奨表記   | 避けるべき表記   | 使用場面         |
| ---------- | ---------------- | ---------------- |
| 登録       | 登録する、Submit | 新規登録ボタン   |
| 変更       | 更新、編集、修正 | 情報変更ボタン   |
| 削除       | 消去、取り消し   | 削除ボタン       |
| 検索       | サーチ、探す     | 検索ボタン       |
| 選択       | 選ぶ、チェック   | 選択ボタン       |
| OK         | 決定、OK         | 確定ボタン       |
| キャンセル | 取り消し、戻る   | キャンセルボタン |

### 状態・ステータス関連

| 推奨表記 | 避けるべき表記       | 使用場面       |
| -------- | -------------------- | -------------- |
| 完了     | 終了、済み           | ステータス表示 |
| 処理中   | 実行中、ローディング | 処理状態表示   |
| エラー   | 失敗、NG             | エラー表示     |
| 必須     | 必須入力、必要       | 必須項目マーク |

### メッセージ文言

#### 確認メッセージ
- **推奨**:「〜しますか？」
- **例**:「この内容で登録しますか？」

#### 完了メッセージ
- **推奨**:「〜が完了しました。」
- **例**:「登録が完了しました。」

#### エラーメッセージ
- **推奨**:「〜できませんでした。〜してください。」
- **例**:「ログインできませんでした。入力内容を確認してください。」

### 単位・数値表記

| 項目 | 推奨表記       | 例            |
| ---- | -------------- | ------------- |
| 日付 | YYYY年MM月DD日 | 2024年3月15日 |
| 時刻 | HH:MM          | 14:30         |

## 特記事項

### カタカナ表記のルール
- 外来語は原則カタカナ表記
- 長音符号「ー」は省略しない（例：ユーザー、サーバー）

### 記号の使用
- 句読点は「、」「。」を使用
- カッコは全角「（）」を基本とする
- 数字は半角を使用

### アクセシビリティへの配慮
- 色だけに依存しない情報伝達
- 適切な代替テキストの設定
- 十分なコントラスト比の確保

# UI 表記規約

## 概要

本ドキュメントは、UIデザインにおける表記の統一性を保つための規約です。ユーザー体験の一貫性を確保し、表記揺れによる混乱を防ぐことを目的とし、同一画面・同一機能では必ず同じ表記を使用します。

## Protocol：全角と半角の境目

最優先で守るルール。本ドキュメント・仕様書・コード内のユーザー向け文言・コミットメッセージ等、本プロジェクトに関わるすべての日本語テキストに適用される。

- 全角文字と半角文字の境目にスペースを入れない（例：`microCMS管理画面`、`URLパラメーター`、`Cookieの挙動`、`有効期限1時間`）
- 半角文字どうしの間にはスペースを入れる（例：`Next.js App Router`）
- バッククォートで囲ったコードトークンも半角として扱う（例：`interview`コンテンツ）

## 基本原則

- 文体: 口語（です・ます調）を基本とします
- トーン: 親しみやすく、専門用語は最小限にします
- 表記揺れの防止: 同一画面・同一機能では必ず同じ表記を使います

## 主要な表記規約

### アクション

| 推奨表記   | 避ける表記       | 使用場面         |
| ---------- | ---------------- | ---------------- |
| 登録       | 登録する、Submit | 新規登録ボタン   |
| 変更       | 更新、編集、修正 | 情報変更ボタン   |
| 削除       | 消去、取り消し   | 削除ボタン       |
| 選択       | 選ぶ、チェック   | 選択ボタン       |
| OK         | 決定             | 確定ボタン       |
| キャンセル | 取り消し、戻る   | キャンセルボタン |

### 状態・ステータス

| 推奨表記 | 避ける表記           | 使用場面       |
| -------- | -------------------- | -------------- |
| 完了     | 終了、済み           | ステータス表示 |
| 処理中   | 実行中、ローディング | 処理状態表示   |
| エラー   | 失敗、NG             | エラー表示     |
| 必須     | 必須入力、必要       | 必須項目マーク |

### メッセージ文言

- 確認:「〜しますか？」（例:「この内容で登録しますか？」）
- 完了:「〜が完了しました。」（例:「登録が完了しました。」）
- エラー:「〜できませんでした。〜してください。」（例:「ログインできませんでした。入力内容を確認してください。」）

### 数値・単位

| 項目 | 推奨表記       | 例            |
| ---- | -------------- | ------------- |
| 日付 | YYYY年MM月DD日 | 2024年3月15日 |
| 時刻 | HH:MM          | 14:30         |

## 特記事項

- 外来語は原則カタカナ。長音符号「ー」は省略しません（例: ユーザー、サーバー）。
- 句読点は「、」「。」。カッコは全角「（）」を基本とします。数字は半角を使います。
- 色だけに依存しない情報伝達、適切な代替テキスト、十分なコントラスト比を確保します。

# 開発用ネーミング辞書

## 利用者呼称

| 日本語     | 英語（コード） | 説明                           | 使用場面             |
| ---------- | -------------- | ------------------------------ | -------------------- |
| プレイヤー | player         | ゲームを遊ぶエンドユーザー     | ドキュメント、仕様書 |
| 運用者     | operator       | イベントを運営・管理する担当者 | ドキュメント、仕様書 |

## 謎解き用語

| 日本語       | 英語（コード）                                    | 説明                                                                                 | 使用場面         |
| ------------ | ------------------------------------------------- | ------------------------------------------------------------------------------------ | ---------------- |
| 通常謎       | mission                                           | 通常謎関連                                                                           | ルーティング、UI |
| 大謎         | grandMission (API識別子), grand-mission (URLパス) | 大謎関連。API識別子はlowerCamelCase、URLパスはkebab-case                             | ルーティング、UI |
| 手がかり     | clue                                              | リアルロケーションへ設置されている、解答に不可欠または正答の推察を補助するための情報 | ルーティング、UI |
| チェックイン | checkIn                                           | リリアルロケーションの緯度経度を計測し、しきい値範囲内外で正誤判定する出題形式       | ルーティング、UI |

## フロントエンド命名規則

### コンポーネント名
- **形式**: パスカルケース
- **例**: `TimeTrackingGrid`, `CaseSalesTable`, `MonthlySalesChart`

### CSS クラス名
- **形式**: キャメルケース（CSS Module使用）
- **例**: `header`, `card`, `button`
- **コンディション**: 先頭にハイフン2つ `--isActive`, `--isDisabled`
- **原則**: 可読性重視の意味ベースな短いネーミングを奨励

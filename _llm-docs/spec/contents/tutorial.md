# オンボーディング

## 概要

初回起動時に表示される導入フロー。オープニングストーリー、プロフィール登録、遊び方説明の流れで構成される。

## フロー構成

1. **オープニングストーリー** (`/story/opening`)
   - ストーリー表示
   - 読了後、プロフィール登録へ遷移

2. **プロフィール登録** (`/profile`)
   - ユーザー情報入力
   - プロフィール保存後、HOME画面へ遷移

3. **遊び方説明** (`/home` でモーダル自動表示)
   - 初回HOME訪問時に自動表示
   - モーダルを閉じるとオンボーディング完了

## 機能要件

### オンボーディング完了状態管理

- **初期遷移判定**
  - アプリ起動時にプロフィールの有無で遷移先を判定
  - プロフィール未登録：オープニングストーリー（`/story/opening`）へ遷移
  - プロフィール登録済み：HOME画面（`/home`）へ遷移
  - HOME画面で`isOnboardingCompleted`がfalseの場合、遊び方説明モーダルを自動表示

- **中間状態**
  - プロフィール登録完了後、HOME画面へ遷移
  - この時点では`isOnboardingCompleted`はまだ`false`
  - HOME画面で遊び方説明モーダルが自動表示される
  - プロフィール登録は完了しているが、オンボーディングは未完了の状態

- **状態更新**
  - 遊び方説明モーダルを閉じた際に完了フラグを立てる
  - Redux Storeで `isOnboardingCompleted` を `true` に更新
  - この時点でオンボーディング完了

### プロフィール登録

- **入力項目**
  - 性別: 女性/男性/無回答（ラジオボタン）
  - 年代: 10代以下〜60代以上（プルダウン）
  - 地域: 47都道府県（プルダウン）
  - 交通手段: 新幹線/在来線/自家用車/バス/その他（プルダウン）

- **バリデーション**
  - 全項目必須入力
  - 全て入力しないと「登録する」ボタンは活性化しない

### 遊び方説明

- **表示制御**
  - HOME画面でモーダルとして表示する
  - `isOnboardingCompleted`がfalseの場合に自動で表示する
  - 初回以降はメニューから`/play-guide`で再表示可能

- **完了処理**
  - モーダルを閉じる際に`setOnboardingCompleted()`を実行する
  - Redux Storeの状態を更新してオンボーディング完了

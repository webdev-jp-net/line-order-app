# 技術構成仕様書

## アーキテクチャ概要

### システム構成

- **フロントエンド**: React SPA (Single Page Application)
- **バックエンド**: REST API
- **ホスティング**: AWS Amplify
- **CMS**: microCMS
- **認証基盤**: LINE LIFF (LINE Front-end Framework)

### 開発環境

- **開発手法**: スキーマ駆動開発
- **API連携**: OpenAPI仕様書からの自動生成
- **品質保証**: husky+lint-stagedによるコミット時自動チェック
- **インフラ管理**: Infrastructure as Code（AWS CDK）

## フロントエンド技術スタック

### コア技術

- **フレームワーク**：React 19.x（最新安定版）
- **ビルドツール**：Vite v6.x（最新版）
- **言語**：TypeScript 5.8.x
- **スタイリング**：CSS Modules
- **状態管理**：Redux Toolkit（RTK）
- **データフェッチング**: RTK Query
- **テストフレームワーク**: Vitest

### 詳細仕様

#### フロントエンドフレームワーク

##### React

- **バージョン**：19.x（※開発状況により最新安定版へ適宜更新）
- **理由**：
  - コンポーネントベースの構造により保守性が高い
  - Viteにより、開発速度とビルド性能が大幅に向上
  - AWS Amplify Hostingとの親和性が高く、CSR前提で構築しやすい
  - v19.xは最新のLTS安定版。非同期処理・アーキテクチャ改善済。長期運用にも安心。
- **レンダリング方式**: CSR（Client Side Rendering）
- **コンポーネント設計**: 関数コンポーネント+Hooks
- **メタタグ管理**: @dr.pogodin/react-helmet v2.0.4（React19対応）

#### ビルド環境

##### Vite

- **用途**: 高速な開発サーバーとビルド
- **設定項目**:
  - HMR（Hot Module Replacement）
  - TypeScriptサポート
  - CSS Modules設定
  - 環境変数管理

#### 型安全性

##### TypeScript

- **バージョン**：5.8.x（※最新安定版へ適宜更新）
- **理由**：
  - 保守性・開発生産性の向上（補完・リファクタリングが容易）
  - APIレスポンス・状態構造を厳密に定義可能
  - 安定性とライブラリ互換性が高い最新の安定版。破壊的変更も少なく移行が容易
  - v5.8.xは長期運用×最新機能×安定性のバランスが良く、今後のライブラリ更新にも耐えうる
- **設定方針**: strict mode有効
- **型定義**: OpenAPIスキーマから自動生成

### スタイリング

#### CSS Modules

- **命名規則**：キャメルケース（CSS Modules使用）
- **スコープ**：コンポーネントローカル
- **PostCSS**：必要に応じて導入

#### CSSリセット

- **パッケージ**: modern-css-reset v1.4.0
- **用途**: ブラウザのデフォルトスタイルをリセット

### 状態管理・データフェッチング

#### Redux Toolkit (RTK)

- **用途**：アプリケーションの状態管理
- **理由**：
  - Redux公式の推奨ツールで、標準化されており信頼性が高い
  - 過去の使用実績が豊富で、開発メンバー間の認識を揃えやすい
  - 今後、認証状態やアプリ全体のUI状態、ユーザー設定などを管理する要件が発生する可能性があるため、RTKを中心に構成しておくことで、柔軟かつスケーラブルな運用が可能
  - RTK Queryと併用することで、API通信と状態をRedux上で一貫して扱うことができ、設計と実装の一体感が得られる
- **スライス設計**：機能単位で分割

#### RTK Query + @rtk-query/codegen-openapi

- **用途**: API通信の管理、自動フェッチ、キャッシュ制御、OpenAPI定義からの自動生成
- **理由**:
  - Redux Toolkitと統一された構成でAPI状態を効率的に管理できる
  - 自動キャッシュやインバリデーション機能によりAPI呼び出しが最適化される
  - `@rtk-query/codegen-openapi`によってOpenAPI仕様の更新とクライアントコード生成が連動し、保守が容易となる
  - 型安全なAPIクライアントを自動生成できるため、実装ミスが起きにくい
- **キャッシュ戦略**: タグベース無効化

### UI開発環境

#### Storybook

- **用途**: UIコンポーネントの独立開発
- **対象**: パーツコンポーネント（汎用:src/components/、ページ固有:src/pages/*/components/）
- **統合**: Vite設定を共有

### テスト

#### Vitest

- **単体テスト**: コンポーネント、ユーティリティ
- **統合テスト**: API連携部分

## バックエンド技術スタック

### コア技術

- **ランタイム**: AWS Lambda
- **データベース**: Amazon DynamoDB
- **インフラ管理**: AWS CDK v2
- **言語**: Node.js
- **API仕様**: OpenAPI 3.0（YAML）
- **モックサーバー**: Prism（Stoplight）
- **デプロイツール**: AWS CLI v2

### 詳細仕様

#### サーバーレスアーキテクチャ

##### AWS Lambda

- **用途**: APIバックエンドの実行環境
- **特徴**:
  - サーバーレスでスケーラブル
  - イベント駆動型の処理
  - コールドスタート対策済み
  - 自動スケーリング
- **構築方法**: AWS CDKのNodejsFunctionを利用

#### インフラ管理

##### AWS CDK（Cloud Development Kit）

- **バージョン**: v2（最新安定版）
- **用途**: インフラの構築・デプロイ管理
- **言語**: TypeScript
- **特徴**:
  - Infrastructure as Code
  - 型安全なインフラ定義
  - 再利用可能なコンストラクト
  - マルチ環境対応（dev/prod）
- **デプロイ方法**: `cdk deploy` コマンド
- **環境管理**: 環境変数による設定切り替え

#### データベース

##### Amazon DynamoDB

- **用途**: NoSQLデータベース
- **特徴**:
  - サーバーレスでフルマネージド
  - 高可用性・高パフォーマンス
  - 自動スケーリング
  - ACIDトランザクション対応
- **アクセス方法**: AWS SDK for JavaScript（v3）
- **設計パターン**: Single Table Design

### 開発・テスト環境

#### API モックサーバー

##### Prism (Stoplight)

- **用途**: OpenAPI仕様書ベースのモックサーバー
- **特徴**:
  - OpenAPI（YAML）からの自動生成
  - リアルタイムでのAPIレスポンス検証
  - ローカル開発環境でのAPIテスト
  - フロントエンド開発の並行作業支援
- **実行環境**: ローカル開発環境

#### デプロイツール

##### AWS CLI

- **バージョン**: v2（最新安定版）
- **用途**: AWSリソースの操作・デプロイ
- **機能**:
  - ローカルからのAWSリソース操作
  - CDKデプロイの実行
  - 環境設定の管理

  - 認証情報の管理

### 外部サービス統合

#### LINE プラットフォーム連携

##### LIFF（LINE Front-end Framework）

- **用途**: LINEアプリとのAPI連携
- **設定**: チャネルIDを環境変数で管理
- **機能**:
  - LINEユーザー認証
  - LINEプロフィール情報取得
  - LINE内ブラウザでの動作
  - LINEメッセージングAPI連携

## 開発環境要件

### 必須環境

- **Node.js**: 18.x以上
- **パッケージマネージャー**: pnpm（プロジェクトで統一）

### 推奨開発ツール

- **IDE**: VS Code
- **拡張機能**:
  - ESLint
  - Prettier
  - TypeScript

### 品質管理ツール

- **husky**: v9.1.7 - Gitフックの管理
- **lint-staged**: コミット時の段階的lint実行
- **対象ファイル**:
  - `src/**/*.{js,jsx,ts,tsx}` - ESLintによるJavaScript/TypeScriptチェック
  - `src/**/*.{css,scss}` - StylelintによるCSS/SCSSチェック
- **動作**: `git commit`時に自動的に該当ファイルのチェックを実行

## バージョン管理方針

- フレームワーク・パッケージのバージョンは開発の都合や最新安定版のリリースがあった場合に適宜更新
- メジャーバージョン更新は慎重に検討
- セキュリティアップデートは即座に適用
- 開発中は最新安定版を追跡
- package-lock.json/yarn.lockのコミット必須
